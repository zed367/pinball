# pinball_kuji — 핀볼(플린코 변형) 경품 추첨 프로토타입 핸드오프

> 이 문서 + 이 프로젝트 폴더(zip) 하나만 있으면 다른 환경의 Claude Code에서도 이어서 작업할 수 있습니다.

---

## 1. 개요

- **무엇**: "쿠지"(경품 추첨) 컨셉의 핀볼/플린코 변형 프로토타입. 플런저를 당겨 공을 발사하면 못밭을 튕기며 내려가 하단 등수 칸에 들어가는 방식
- **형태**: 가로형(landscape), 1연(공 1개)짜리 발사 — 카드팩 매대 방식(`vibe_kuji`, 같은 workspace 상위 폴더)과는 다른 별도 컨셉 프로토타입
- **기술 스택**: Vite + Vanilla JS + [`matter-js`](https://brm.io/matter-js/)(2D 물리 엔진) + `<canvas>` 렌더링. 프레임워크(React 등) 없음
- **핵심 설계 원칙(코드 주석에 반영됨)**:
  - 등수 결정은 가중치 RNG로 직접 뽑지 않는다 — 재고에 맞춰 하단 칸에 등수를 배정해두고, **공이 못밭을 물리적으로 튕기며 어느 칸에 들어가는지가 곧 결과**다
  - 세게 당길수록 공은 힘차게 나가지만, 어느 칸에 들어가는지는 발사 세기만으로 정해지지 않음(못밭 6줄을 지나며 생기는 카오스적 튕김이 실제 결과를 좌우) — "세기로 결과를 고를 수 없다"를 유저에게도 문구로 안내 중
  - 1등 슬롯은 가장 좁고, 2등·3등·4등 슬롯은 순서대로 넓어짐(폭 비율 1:2:3:4). 판 새로고침 시 등수 슬롯 위치 전체를 다시 섞음

## 2. 실행 방법

```bash
npm install
npm run dev
```

터미널에 안내되는 주소(`http://localhost:5173` 등)를 브라우저로 열면 됩니다.

플레이: 우측 플런저 레인(캔버스 우측 세로 파이프)을 아래로 드래그(포인터 다운 → 아래로 끌기 → 놓기)하면 발사됩니다. `MIN_PULL_RATIO_TO_FIRE`(0.15) 이상 당겨야 발사됩니다.

## 3. 파일 구조

```
pinball_kuji/
├─ index.html
├─ src/
│  ├─ main.js          렌더 루프 · 플런저 드래그 인터랙션 · 발사/착지 처리 · UI
│  ├─ physicsWorld.js   matter-js 물리 월드(벽·레인·못밭·슬롯 센서/뚜껑) 구성
│  ├─ pinballEngine.js  물리와 분리된 순수 로직 — 재고, 칸 배정, 당첨 판정
│  └─ style.css
└─ public/              (현재 비어있음)
```

## 4. 핵심 로직 흐름

1. `pinballEngine.js`의 `createDemoPrizes()`로 데모 재고 생성 (1등 1개 / 2등 3개 / 3등 6개 / 4등 10개)
2. `buildSlotSequence()`로 하단 칸 배치를 한 번 뽑고, 등수별 폭 비율(1:2:3:4)에 맞춘 슬롯 경계를 계산. 판 새로고침 시 위치와 폭을 함께 다시 섞음
3. `physicsWorld.js`의 `createPhysicsWorld()`가 못밭(6행) · 레인(곡선 파이프) · 등수별 폭의 슬롯 센서/구분벽 · 재고 0인 칸을 막는 "뚜껑"(lid) 콜라이더를 구성
4. `main.js`가 플런저 드래그로 `pullRatio`(0~1)를 계산해 `launchBall(pullRatio)` 호출 → 속도는 `MIN_LAUNCH_SPEED`~`MAX_LAUNCH_SPEED` 사이 비례, x축엔 아주 작은 랜덤 편차만 줘서 매번 다른 결과가 나오게 함
5. 공이 슬롯 센서에 닿으면(`collisionStart`) `onLanding(slotIndex)` → `resolveLanding()`이 해당 등수 재고를 1 차감하고 결과 반환
6. 당첨 후 `buildSlotLayout()`을 재호출해 소진된 등수의 칸을 다시 "마감"으로 막고, 재고 패널 갱신

## 5. 알려진 디자인 결정 / 주의사항 (코드 주석에서)

- 플런저 손잡이 애니메이션: 스프링(코일) 모양 자체는 고정해서 그린다. 당김에 따라 코일이 압축되는 애니메이션을 넣었더니 "당기는데 코일 아래쪽이 위로 올라가는 것처럼 보여 헷갈린다"는 문제가 있어서, 대신 손잡이(원)+막대만 아래로 움직이는 방식으로 "당기는 느낌"을 표현 (`main.js` `drawPlunger()`)
- 레인 콜라이더는 각진 사각형 조각들의 연결이지만, 화면엔 `getLaneTubePoints()`의 좌표를 곡선 하나로 이어그려 매끈한 금속 파이프처럼 보이게 함 — 콜라이더 렌더와 화면 렌더가 분리되어 있음
- 필드-레인 사이 틈을 막는 벽이 없으면 마지막 칸(4등) 옆으로 공이 새서 슬롯 센서를 못 만나 영영 착지 판정이 안 나는 버그가 있었음 → `physicsWorld.js`의 필드/레인 사이 벽이 이를 막음
- 고정 발사 제한시간은 두지 않음. 대신 공의 실제 속도가 임계값 아래인 상태가 2초 이상 지속될 때만 끼임으로 판정해 공을 정리하고 재발사를 허용함 (물리 엣지케이스 방어)
- `engine.gravity.y = 0.92`(기본 1보다 낮음): 발사된 공이 좀 더 오래 체공해야 멀리 나가서, 속도를 아무리 올려도 중력이 강하면 금방 떨어지는 문제를 완화
- `positionIterations`/`velocityIterations`을 기본보다 높여 터널링(얇은 벽을 뚫고 지나가는 물리 버그) 방지, 곡선 레일 이음매마다 원 콜라이더를 덧대 틈으로 새는 것도 방지
